## 📋 API Endpoints Summary

### Module 1: Data Ingestion (6 APIs)

| # | Method | Endpoint | Description |
|----|--------|----------|-------------|
| 1 | GET | `/api/v1/ingestion/available-dates` | Get all available trade dates |
| 2 | POST | `/api/v1/ingestion/download` | Download & parse NSE data |
| 3 | GET | `/api/v1/ingestion/history` | Get download history |
| 4 | GET | `/api/v1/ingestion/status/{date}` | Check data for a date |
| 5 | DELETE | `/api/v1/ingestion/{date}` | Delete data for a date |
| 6 | GET | `/api/v1/ingestion/summary` | Get statistics |

### Module 2: Data Explorer (6 APIs)

| # | Method | Endpoint | Description |
|----|--------|----------|-------------|
| 1 | GET | `/api/v1/stocks` | Search with filters & pagination |
| 2 | GET | `/api/v1/stocks/dates` | Get available dates |
| 3 | GET | `/api/v1/stocks/{symbol}` | Get stock detail |
| 4 | GET | `/api/v1/stocks/sectors` | Get all sectors |
| 5 | GET | `/api/v1/stocks/top-gainers` | Get top gainers |
| 6 | GET | `/api/v1/stocks/top-losers` | Get top losers |

## 🎯 Example Requests

### 1. Download NSE Data

### 3. Get Top Gainers

## 📅 Important: Date Format

⚠️ **Different date formats for different modules:**

### Module 1 (Data Ingestion)
- Format: `yyyy-MM-dd`
- Example: `2025-01-15`
- Used in: POST download, DELETE, GET status

### Module 2 (Data Explorer)
- Format: `dd-MM-yyyy`
- Example: `15-01-2025`
- Used in: Search, Top gainers/losers, Stock detail

## 💡 Testing Workflow

### Recommended Test Sequence:

1. **Check Available Dates**
   ```bash
   curl "http://localhost:8080/api/v1/ingestion/available-dates" | jq '.'
   ```

2. **Download Data** (if needed)
   ```bash
   curl -X POST "http://localhost:8080/api/v1/ingestion/download" \
     -H "Content-Type: application/json" \
     -d '{"tradeDate": "2025-01-15", "overwrite": false}'
   ```

3. **Verify Download**
   ```bash
   curl "http://localhost:8080/api/v1/ingestion/status/2025-01-15" | jq '.'
   ```

4. **Get Available Stock Dates**
   ```bash
   curl "http://localhost:8080/api/v1/stocks/dates" | jq '.'
   ```

5. **Search Stocks**
   ```bash
   curl "http://localhost:8080/api/v1/stocks?date=15-01-2025&page=0&size=10" | jq '.'
   ```

6. **Get Top Gainers/Losers**
   ```bash
   curl "http://localhost:8080/api/v1/stocks/top-gainers?date=15-01-2025&limit=5" | jq '.'
   curl "http://localhost:8080/api/v1/stocks/top-losers?date=15-01-2025&limit=5" | jq '.'
   ```

## 🐛 Troubleshooting

| Issue | Solution |
|-------|----------|
| "Connection refused" | Ensure app is running on `localhost:8080` |
| "Invalid date format" | Check format: Module 1 uses `yyyy-MM-dd`, Module 2 uses `dd-MM-yyyy` |
| "404 Not Found" | Verify endpoint URL and HTTP method |
| "No data returned" | Download data first using POST `/api/v1/ingestion/download` |
| "JSON parse error" | Ensure response is valid JSON using `jq` |
