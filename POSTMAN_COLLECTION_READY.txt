## 🔑 Important Notes

### Date Format
⚠️ Two different date formats are used:

**Module 1 (Data Ingestion):** `yyyy-MM-dd`
- Example: `2025-01-15`
- Used in: Download, Delete, Status check

**Module 2 (Data Explorer):** `dd-MM-yyyy`
- Example: `15-01-2025`
- Used in: Search, Top gainers/losers, Stock detail

### Configuration
Update in Postman:
- **Variables** → **baseUrl** → Your API URL

### Requirements
- Application must be running on `localhost:8080`
- Database must be populated with sample data
- Postman or cURL for testing

## 📊 EXAMPLE API CALLS

### Example 1: Search Stocks
```bash
curl "http://localhost:8080/api/v1/stocks?date=15-01-2025&minPrice=100&maxPrice=5000&page=0&size=50" | jq '.'
```

### Example 2: Get Top Gainers
```bash
curl "http://localhost:8080/api/v1/stocks/top-gainers?date=15-01-2025&limit=10" | jq '.'
```

### Example 3: Download Data
```bash
curl -X POST "http://localhost:8080/api/v1/ingestion/download" \
  -H "Content-Type: application/json" \
  -d '{"tradeDate": "2025-01-15", "overwrite": false}'
```

### Example 4: Get Stock Detail
```bash
curl "http://localhost:8080/api/v1/stocks/INFY?date=15-01-2025" | jq '.'
```

## 🛠️ TROUBLESHOOTING

| Problem | Solution |
|---------|----------|
| "Connection refused" | Ensure app is running: `gradle bootRun` |
| "Invalid date format" | Check format: Module 1 uses `yyyy-MM-dd`, Module 2 uses `dd-MM-yyyy` |
| "Empty response" | Download data first: POST `/api/v1/ingestion/download` |
| "Import fails in Postman" | Verify file is valid JSON |
| "404 Not Found" | Check endpoint URL spelling |
